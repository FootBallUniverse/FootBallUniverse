・C#標準のコメントについて
C#では関数などの前に / を3回連続で入力すると自動的にコメントが追加される。
例：
/// <summary>
/// 
/// </summary>
void GetTriggerKey(){ 
}

※このコメントは使いずらいので基本的に関数の前のコメントは以下の物を使う。

▼関数について！

//----------------------------------------------------------------------
// XBOXのコントローラーのキートリガー取得
//----------------------------------------------------------------------
// @Param	test ～～の値	test2 ～～の値			
// @Return	取得したキー
// @Date	2014/10/7/9:30  @Update 2014/10/7/9:30  @Author T.Kawashita      
//----------------------------------------------------------------------
int GetTriggerKey(uint key){
	
}

１．上の部分はその関数が何をするかを書く。
２．@Paramの部分は引数を書く。例は上の通り。複数ある場合は複数書く。2行になってもOK
３．@Returnの部分は戻り値を書く。例は上の通り。
４．@Dateの部分は作成した日付を書く。例は上の通り。
５．@Updateの部分は更新した日付を書く。例は上の通り。
６．@Authorは作成した人の名前を書く。例は上の通り

※基本的にUnityの標準のStartやUpdateメソッドの部分は変えなくてもよい。
というか作成した関数以外はコメントなくてもOK.

▼変数やその他について

変数についてはわかりにくいものなどはコメント化する。
変数は名前で判断できるものにしておいてください。

その他のクラスなどは自己判断で。
if や forについても同様。
